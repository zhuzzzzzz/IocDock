# IocDock

## 系统部署指南

### 1. 安装工具(git)

**推荐使用 Ubuntu 操作系统**

#### 从 git 安装

```shell
# 在管理主机拉取代码以安装部署管理系统
git clone https://github.com/zhuzzzzzz/IocDock.git

# 安装依赖的 python 包
cd IocDock
pip install -r requirments.txt
# 若提示 python 包由系统包管理器管理, 可分别执行 apt install python-package_name 完成安装

# 安装工具, 并重启终端使配置生效, 工具默认安装在 /opt/IocDockHome/IocDock/ 目录下
# 注意 安装将创建 用户id=9981 组id=9981 的系统用户iocdock, 请确保集群内其他主机均未占用相关编号!
sudo ./install.sh

# 多用户配置(为本机其他用户配置工具的执行权限)
sudo usermod -aG iocdock $USER
sudo newgrp iocdock
```

#### 从 git 更新

```shell
# 提前保存用户的自定义配置, 然后执行这条命令将代码恢复至初始状态
git reset --hard

# 更新仓库并拉取最新代码
git remote update
git pull

# 执行安装
sudo ./install.sh
```

### 2. 构建集群环境

#### 2.1 基于 ansible role 自动化搭建 swarm 集群

0. 切换至工具安装目录, 在配置文件 `settings.py` 中定义集群栈名称 `PREFIX_STACK_NAME`.

1. 修改配置文件 `settings.py` 中 `Ansible 配置` 部分, 定义所有集群节点的主机名称及对应ip, 远程用户, 创建用户, 密码(密码可暂不设置), swarm 初始网卡等参数.

2. 生成 ansible 主机清单文件

   ```shell
   cd $IOCDOCK_MANAGER_PATH
   # 生成 ansible 主机清单文件
   IocManager cluster --gen-inventory-files
   ```

3. 配置集群根证书

    ```shell
    IocManager make-certs -- --root
    # 使用 --force 覆盖已存在的证书
    IocManager make-certs -- --root --force

    # 查看已有证书
    IocManager manage-certs -- list

    # 配置集群节点信任根证书
    IocManager cluster --set-up-root-cert
    ```

4. 调用 playbook 自动初始化集群环境

    ```shell
    # 调用后, 工具将执行以下步骤: 创建远程用户, 设置 ssh 连接, 设置基础运行环境, 安装 docker 依赖并构建 swarm 集群
    $ IocManager cluster --set-up-cluster

    # 此时已安装 docker, 需重新启动 IocDockServer 服务
    $ systemctl restart IocDockServer.service

    # 支持分步执行, 仅创建远程用户
    $ IocManager cluster --create-remote-user
    # 分步执行, 仅设置 ssh 连接
    $ IocManager cluster --set-up-ssh-connection
    # 分步执行, 仅设置基础运行环境
    $ IocManager cluster --set-up-basic-environment
    # 分步执行, 仅构建 swarm 集群
    $ IocManager cluster --set-up-swarm
    ```

5. 验证集群环境

    ```shell
    # 运行以下命令显示集群信息, 若无报错则表示集群搭建成功
    $ IocManager cluster --ping 
    $ IocManager swarm --show-digest
    $ IocManager swarm --show-nodes --detail

    # ansible 主机清单文件位于 imtools/ansible/inventory/ 目录下, 可作为执行 ansible 操作的主机清单文件
    # 使用此清单执行一般权限的 ansible 操作
    ansible all -m ping -i imtools/ansible/inventory/ -k
    # 使用此清单执行需要 root 权限的 ansible 操作
    ansible all -m ping -i imtools/ansible/inventory/ -bkK
    ```

#### 2.2 配置集群 NFS 服务

集群使用 NFS 来存储和共享系统内部公共数据.

**由于 swarm 的编排调度对容器服务部署文件的解析和插值均发生在命令执行节点, 因此要求集群内所有节点具有与管理节点相同的路径结构(在`/home/user_name/docker/`目录下).**

1. 配置 NFS server

    ```shell
    sudo apt install nfs-kernel-server

    sudo mkdir -p /NFS/IocDock-data
    sudo mkdir -p /NFS/registry-data
    sudo chown iocdock:iocdock -R /NFS
    sudo chmod 777 -R /NFS

    $ sudo vi /etc/exports
    # /NFS/IocDock-data 192.168.1.0/24(rw,sync,all_squash,no_subtree_check)
    # /NFS/registry-data 192.168.1.0/24(rw,sync,all_squash,no_subtree_check)
    
    sudo exportfs -a
    ```

2. 修改配置文件 `settings.py` 中 NFS 相关配置项, 登记集群内的 NFS 目录挂载信息

    ```python
    MOUNT_DIR_NFS_MOUNT_SRC = "192.168.1.50:/home/iocdock/NFS/IocDock-data"
    REGISTRY_NFS_MOUNT_SRC = "192.168.1.50:/home/iocdock/NFS/registry-data"
    ```

### 3. 部署预置的集群基础设施服务

#### 3.1 部署 registry

1. 修改配置文件 `settings.py` 中 `Registry 配置` 部分, 定义初始 registry 节点地址及镜像仓库的账号密码

2. 生成 registry 部署文件

    ```shell
    # 生成服务证书
    IocManager make-certs -- --server registry [--force]

    # 生成 registry 部署文件(可先设置registry, 其他服务暂时回车跳过)
    IocManager swarm --gen-built-in-services
    ```

3. 为需要运行 registry 的节点打上 swarm 标签(包括本机), 随后更新目录设置. 服务采用 NFS 共享存储, 因此需将所有运行 registry 服务主机的
   registry-data 目录挂载至对应 NFS 公共目录

    ```shell
    docker node update node_name --label-add registry=true 
    IocManager cluster --set-up-file-and-dir
    ```

4. 部署 registry 服务

    ```shell
    # 准备本地镜像
    cd imsrvs/registry/scripts
    ./prepare-images.sh

    # 部署 registry 服务
    IocManager service registry --deploy

    # 配置节点 DNS 解析条目, 并登录镜像仓库
    IocManager cluster --registry-login

    # 上传服务镜像
    IocManager cluster --prepare-service-images

    # 等待其余 registry 服务上线(若一直提示没有镜像, 请停止并重新部署 registry 服务)
    ```

5. 准备 EPICS base 镜像及 IOC 镜像

    ```shell
    IocManager cluster --prepare-ioc-images
    ```

#### 3.2 部署全局服务(alloy, cAdviosr, nodeExporter, iocLogServer, client)

1. 执行命令生成部署文件, 更新目录设置, 然后部署

    ```shell
    $ IocManager swarm --gen-builtin-services 
    $ IocManager cluster --set-up-file-and-dir
    $ IocManager swarm --deploy-global-services
    # alloy 服务地址: 192.168.1.50:12345
    # cAdvisor 服务地址: 192.168.1.50:8080
    # nodeExporter 服务地址: 192.168.1.50:9100
    # iocLogServer 服务地址: 192.168.1.50:7004
    ```

#### 3.3 部署本地服务(prometheus, loki, grafana)

1. 执行命令生成部署文件, 打上节点 swarm 标签, 更新目录设置, 然后部署

    ```shell
    $ IocManager swarm --gen-builtin-services 
    docker node update node_name --label-add prometheus=true 
    docker node update node_name --label-add loki=true 
    docker node update node_name --label-add grafana=true 
    IocManager cluster --set-up-file-and-dir
    ```

2. 部署

    ```shell
     $ IocManager service prometheus --deploy
     # prometheus 服务地址: 192.168.1.50:9090
     $ IocManager service loki --deploy
     # loki 服务地址: 192.168.1.50:3100
     $ IocManager service grafana --deploy
     # grafana 服务地址: 192.168.1.50:3000
    ```

#### 3.4 部署本地服务(alertManager)

1. 设置 `IMConfig.py` 中 smtp 发件邮箱相关信息. 密码可以在其中设置密码或使用密码文件来实现

    ```yaml
     smtp_from: zhujunhua@mail.iasf.ac.cn
     smtp_smarthost: smtp.qiye.aliyun.com:465
     smtp_auth_username: zhujunhua@mail.iasf.ac.cn
     smtp_auth_password_file: /etc/alertmanager/smtp_password
     smtp_require_tls: false
    ```

2. 设置 `alertManager/config/alertManager-config.yaml` 以配置报警分级路由, 邮件接收分组等功能.

3. 打上节点标签, 更新目录设置, 执行部署

    ```shell
     $ IocManager swarm --gen-builtin-services
     $ docker node update node_name --label-add alertmanager=true 
     $ IocManager cluster --set-up-file-and-dir
     $ IocManager service alertManager --deploy
     # 192.168.1.50:9093
    ```

#### 4.2 部署生产环境 IOC 项目

如何开发 IOC 项目, 见后文 [IOC 项目的开发与部署](#ioc-development-and-deployment).

IOC 项目创建到部署通常的工作流程如下:

```shell
# 创建IOC项目
$ IocManager create ioc_name

# 为IOC项目添加源文件
$ IocManager exec ioc_name --add-src-file /path/to/dir

# 配置IOC项目
$ IocManager edit ioc_name

# 为IOC项目生成运行文件, 将IOC项目导出至运行目录, 为运行目录内的IOC项目生成swarm部署文件
$ IocManager exec ioc_name --deploy

# 启动IOC的swarm容器服务
$ IocManager Service ioc_name --deploy

# 停止IOC的swarm容器服务
$ IocManager Service ioc_name --remove
```
